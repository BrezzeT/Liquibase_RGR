# РГР: Робота із версійністю баз даних за допомогою Liquibase

## 📋 Опис проекту
У даній роботі продемонстровано використання інструменту **Liquibase** для автоматизації контролю версій структури бази даних **H2**. Проект охоплює повний цикл життєвого циклу міграцій: від початкового налаштування до механізмів відкату змін у випадку помилок.

## 🚀 Основні можливості
- **Декларативний підхід**: Опис структури БД за допомогою універсального формату XML.
- **Складна архітектура**: Реалізовано схему бази даних інтернет-магазину з використанням зовнішніх ключів (Foreign Keys).
- **Контроль стану**: Використання тегів для фіксації стабільних версій схеми.
- **Відкат змін**: Можливість автоматичного повернення до попередніх станів бази.

## 🛠 Перелік використаних команд
* `liquibase status` — перевірка наявності незастосованих змін.
* `liquibase update` — застосування міграцій до бази даних.
* `liquibase tag [name]` — створення мітки поточної версії.
* `liquibase rollback [tag]` — відкат бази до вказаної мітки.

## 📂 Файлова структура
* `liquibase.properties` — параметри підключення до БД (JDBC URL, драйвер, шлях до ченджлогу).
* `changelog.xml` — головний файл проекту, що містить усі `changeSets`.

## ⚙️ Як розгорнути проект локально
1. Встановити **Liquibase CLI**.
2. Переконатися, що файли проекту знаходяться в одній директорії.
3. Виконати команду для ініціалізації та створення таблиць:
   ```bash
   liquibase update
   ```
4. Для перевірки структури можна використати команду генерації документації:
   ```bash
   liquibase db-doc documentation
   ```
